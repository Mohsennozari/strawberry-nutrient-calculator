## 📖 درباره پروژه

این سیستم به کشاورزان و کارشناسان گلخانه کمک می‌کند تا با **۶ اندازه‌گیری ساده** (قطر طوقه، ارتفاع بوته، طول دمبرگ، طول برگ، عرض برگ، اندازه میوه) و ثبت اطلاعات محیطی (دما، وضعیت هوا، EC، pH)، نیاز غذایی دقیق گیاه توت‌فرنگی را دریافت کنند.

**تفاوت این سیستم با روش‌های سنتی:**

| روش سنتی | سیستم تصمیم‌یار |
|:---|:---|
| ❌ جدول‌های ثابت و پیش‌فرض | ✅ محاسبه بر اساس فیزیولوژی واقعی گیاه |
| ❌ تخمین چشمی و تجربه شخصی | ✅ داده‌های علمی معتبر (DRIS, UF/IFAS) |
| ❌ فقط N, P, K | ✅ ۱۳ عنصر غذایی + ۷ نسبت کلیدی |
| ❌ بدون منبع علمی | ✅ هر عدد به منبع خاص ارجاع داده شده |

## 🎯 رسالت سیستم

> **این سیستم فقط و فقط یک «ماشین‌حساب تغذیه» است که می‌گوید:**
>
> **«با توجه به وضعیت فعلی گیاه، به چند ppm از هر عنصر نیاز داری؟»**

### وظایف اصلی:

| کار | توضیح |
|:---|:---|
| **تشخیص مرحله رشد** | بر اساس ۶ ویژگی فیزیولوژیکی و سن بوته |
| **محاسبه ppm عناصر** | ۱۳ عنصر غذایی با استناد به منابع علمی |
| **نمایش وضعیت** | 🟢 بهینه / 🔴 کمبود / 🟠 بیش‌ازحد |
| **نسبت‌های کلیدی** | ۷ نسبت مهم (K:Ca, K:Mg, ...) |
| **EC و pH پیشنهادی** | محدوده‌های بهینه برای محلول غذایی |
| **هشدارها و توصیه‌ها** | اقدامات اصلاحی دقیق و عملی |

### ⚠️ این سیستم مسئول چه چیزهایی نیست:

- ❌ نسخه‌دهی کود (سیستم دیگر)
- ❌ تنظیم تانک‌های آبیاری (سیستم دیگر)
- ❌ مدیریت زه‌آب (سیستم دیگر)
- ❌ زمان‌بندی آبیاری (سیستم دیگر)

## ✨ ویژگی‌ها

### فنی
- **Backend:** Flask (Python) – API برای محاسبات
- **Frontend:** Vue.js 3 + Tailwind CSS – رابط کاربری زیبا و واکنش‌گرا
- **ماژولار:** کد به ۳ بخش مجزا تقسیم شده (داده‌ها، هشدارها، موتور محاسبه)
- **پشتیبانی از RTL:** طراحی شده برای زبان فارسی
- **لاگ‌گیری:** ثبت تمام درخواست‌ها و خطاها در پوشه `logs/`
- **محدودیت نرخ:** محافظت از سرور در برابر درخواست‌های زیاد
- **مستندات API:** مسیر `/api/docs` برای مشاهده مستندات کامل

### کاربردی
- **۱۳ عنصر غذایی:** N, P, K, Ca, Mg, S, Fe, Zn, Mn, Cu, B, Mo, Cl
- **۷ نسبت کلیدی:** K:Ca, K:Mg, Ca:Mg, N:K, K:S, Fe:Mn, B:Ca
- **پشتیبانی از ارقام مختلف:** Albion, Camarosa, Sweet Charlie
- **پیش‌بینی وزن میوه:** با مدل غیرخطی (R² = 0.92)
- **هشدارهای دسته‌بندی‌شده:** بحرانی / هشدار / اطلاع‌رسانی
- **توصیه‌های دقیق:** با ذکر منبع علمی و اولویت اجرا
- **خروجی CSV و کپی جدول:** برای گزارش‌گیری
- **نمایش منابع علمی:** هر عدد به منبع خاص ارجاع داده شده
- **اعتبارسنجی ورودی:** جلوگیری از ورود داده‌های غیرمنطقی
- **تب «درباره محاسبات»:** توضیح کامل هر فیلد و محاسبات پشت صحنه

## 🚀 نصب و راه‌اندازی

### ۱. کلون کردن پروژه

```bash
git clone https://github.com/yourusername/strawberry-nutrient-system.git
cd strawberry-nutrient-system
```

### ۲. راه‌اندازی Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

سرویس Backend روی پورت `5000` اجرا می‌شود:
```
http://127.0.0.1:5000
```

### ۳. راه‌اندازی Frontend

```bash
cd frontend
npm install
npm run dev
```

سرویس Frontend روی پورت `5173` اجرا می‌شود:
```
http://127.0.0.1:5173
```

### ۴. استفاده از مدیریت پروژه (پیشنهادی)

```bash
python manage.py
```

سپس گزینه مورد نظر را انتخاب کنید:
- `1` : اجرای همزمان Backend + Frontend
- `2` : توقف همه سرویس‌ها
- `3` : بکاپ‌گیری از فایل‌ها
- `4` : نمایش وضعیت
- `5` : پاک کردن کش
- `6` : نصب وابستگی‌ها
- `7` : اجرای تست‌ها
- `0` : خروج

### ۵. اجرای تست (اختیاری)

```bash
cd backend
python test_ec_ph.py
```

## 📖 راهنمای استفاده

### گام ۱: اندازه‌گیری (۱۰ دقیقه در هفته)

| فیلد | ابزار | محدوده نرمال |
| :--- | :--- | :--- |
| قطر طوقه | کولیس | ۱۲ – ۱۷ میلی‌متر |
| ارتفاع بوته | خط‌کش | ۲۰ – ۳۰ سانتی‌متر |
| طول دمبرگ | خط‌کش | ۱۰ – ۲۰ سانتی‌متر |
| طول برگ | خط‌کش | ۶ – ۸ سانتی‌متر |
| عرض برگ | خط‌کش | ۴.۵ – ۶.۵ سانتی‌متر |
| اندازه میوه | خط‌کش | ۳ – ۱۰ سانتی‌متر مربع |

### گام ۲: ورود اطلاعات

1. وارد تب **«محاسبه‌گر»** شوید.
2. مقادیر اندازه‌گیری‌شده را در فیلدهای مربوطه وارد کنید.
3. دما و وضعیت هوا را ثبت کنید.
4. (اختیاری) EC و pH را وارد کنید.
5. رقم مورد نظر را انتخاب کنید (Albion / Camarosa / Sweet Charlie).
6. دکمه **«محاسبه نیاز غذایی»** را بزنید.

### گام ۳: دریافت خروجی

سیستم خروجی‌های زیر را نمایش می‌دهد:

1. **مرحله رشد** (رویشی / گلدهی / باردهی / رسیدگی)
2. **۱۳ عنصر غذایی** با مقدار ppm و وضعیت (بهینه / کم / زیاد)
3. **۷ نسبت کلیدی** با وضعیت
4. **EC و pH** با وضعیت و محدوده بهینه
5. **پیش‌بینی وزن میوه** (بر اساس مدل غیرخطی)
6. **هشدارها** (دسته‌بندی شده: بحرانی / هشدار / اطلاع‌رسانی)
7. **توصیه‌های عملی** (با ذکر منبع علمی و اولویت اجرا)
8. **منابع علمی** (نمایش منبع هر عنصر)

### گام ۴: اقدام عملی

- اگر هشدار **بحرانی** (قرمز) است → **فوری اقدام کنید**
- اگر هشدار **هشدار** (زرد) است → **بررسی و پایش کنید**
- اگر هشدار **اطلاع‌رسانی** (آبی) است → **توجه داشته باشید**
- توصیه‌ها را به ترتیب اولویت (بالا / متوسط / کم) اجرا کنید
