## 📖 درباره پروژه

این سیستم به کشاورزان و کارشناسان گلخانه کمک می‌کند تا با **۶ اندازه‌گیری ساده** (قطر طوقه، ارتفاع بوته، طول دمبرگ، طول برگ، عرض برگ، اندازه میوه) و ثبت اطلاعات محیطی (دما، وضعیت هوا)، نیاز غذایی دقیق گیاه توت‌فرنگی را دریافت کنند.

**تفاوت این سیستم با روش‌های سنتی:**
- ❌ نه بر اساس جدول‌های ثابت و پیش‌فرض
- ❌ نه بر اساس تخمین چشمی و تجربه‌ی شخصی
- ✅ بر اساس **پارامترهای فیزیولوژیکی واقعی** گیاه
- ✅ بر اساس **داده‌های علمی معتبر** (DRIS, UF/IFAS, Frontiers 2024)

---

## ✨ ویژگی‌ها

### فنی
- **Backend:** Flask (Python) – API برای محاسبات
- **Frontend:** Vue.js 3 + Tailwind CSS – رابط کاربری زیبا و واکنش‌گرا
- **دیتابیس:** SQLite (برای ذخیره‌سازی تاریخچه و روند رشد)
- **پشتیبانی از RTL:** طراحی شده برای زبان فارسی

### کاربردی
- **۱۳ عنصر غذایی:** N, P, K, Ca, Mg, S, Fe, Zn, Mn, Cu, B, Mo, Cl
- **۷ نسبت کلیدی:** K:Ca, K:Mg, Ca:Mg, N:K, K:S, Fe:Mn, B:Ca
- **پیش‌بینی وزن میوه:** با دقت R² = 0.92
- **هشدارهای هوشمند:** تشخیص کمبود/مسمومیت عناصر
- **خروجی CSV و کپی جدول:** برای گزارش‌گیری

## 🚀 نصب و راه‌اندازی

### ۱. کلون کردن پروژه

```bash
git clone https://github.com/yourusername/strawberry-nutrient-system.git
cd strawberry-nutrient-system
```

### ۲. راه‌اندازی Backend

```bash
cd backend
pip install -r requirements.txt
python app.py
```

سرویس Backend روی پورت `5000` اجرا می‌شود:
```
http://127.0.0.1:5000
```

> **توجه (امنیت):** برای محیط عملیاتی حتماً متغیر محیطی `JWT_SECRET` را با یک مقدار تصادفی و امن تنظیم کنید:
> ```bash
> export JWT_SECRET="یک-کلید-تصادفی-طولانی"
> ```

## 🔐 سیستم ورود (احراز هویت)

برای استفاده از ماشین‌حساب، ابتدا باید یک حساب کاربری بسازید و وارد شوید.

- **ثبت‌نام:** با نام کاربری، ایمیل و رمز عبور (حداقل ۶ کاراکتر) یک حساب بسازید.
- **ورود:** با نام کاربری (یا ایمیل) و رمز عبور وارد شوید.
- توکن ورود به‌صورت JWT صادر و در مرورگر ذخیره می‌شود و درخواست‌های محاسبه‌گر به‌صورت خودکار با آن احراز هویت می‌شوند.
- رمزهای عبور به‌صورت هش‌شده (Werkzeug) در دیتابیس SQLite ذخیره می‌شوند.

### مسیرهای API احراز هویت

| متد | مسیر | شرح | نیاز به توکن |
| :--- | :--- | :--- | :--- |
| `POST` | `/auth/register` | ثبت‌نام کاربر جدید | خیر |
| `POST` | `/auth/login` | ورود و دریافت توکن | خیر |
| `GET` | `/auth/me` | اطلاعات کاربر فعلی | بله |
| `POST` | `/calculate` | محاسبه نیاز غذایی | بله |

ارسال توکن در هدر درخواست‌های محافظت‌شده:
```
Authorization: Bearer <token>
```

### ۳. راه‌اندازی Frontend

```bash
cd frontend
npm install
npm run dev
```

سرویس Frontend روی پورت `5173` اجرا می‌شود:
```
http://127.0.0.1:5173
```

### ۴. اجرای همزمان (پیشنهادی)

```bash
python manage.py
```

سپس گزینه `1` را انتخاب کنید.

## 📖 راهنمای استفاده

### گام ۱: اندازه‌گیری (۱۰ دقیقه در هفته)

| فیلد | ابزار | محدوده نرمال |
| :--- | :--- | :--- |
| قطر طوقه | کولیس | ۱۲ – ۱۷ میلی‌متر |
| ارتفاع بوته | خط‌کش | ۲۰ – ۳۰ سانتی‌متر |
| طول دمبرگ | خط‌کش | ۱۰ – ۲۰ سانتی‌متر |
| طول برگ | خط‌کش | ۶ – ۸ سانتی‌متر |
| عرض برگ | خط‌کش | ۴.۵ – ۶.۵ سانتی‌متر |
| اندازه میوه | خط‌کش | ۳ – ۱۰ سانتی‌متر مربع |

### گام ۲: ورود اطلاعات

1. وارد تب **«محاسبه‌گر»** شوید.
2. مقادیر اندازه‌گیری‌شده را در فیلدهای مربوطه وارد کنید.
3. دما و وضعیت هوا را ثبت کنید.
4. دکمه **«محاسبه نیاز غذایی»** را بزنید.

### گام ۳: دریافت خروجی

سیستم خروجی‌های زیر را نمایش می‌دهد:

1. **مرحله رشد** (رویشی / گلدهی / باردهی / رسیدگی)
2. **۱۳ عنصر غذایی** با وضعیت (بهینه / کم / زیاد)
3. **۷ نسبت کلیدی** با وضعیت
4. **EC و pH** با وضعیت
5. **پیش‌بینی وزن میوه**
6. **هشدارها و توصیه‌های عملی**

### گام ۴: اقدام عملی

- اگر عنصری **قرمز (کمبود)** است → کوددهی آن را افزایش دهید.
- اگر عنصری **نارنجی (بیش‌ازحد)** است → کوددهی آن را کاهش دهید.
- اگر نسبتی خارج از محدوده است → کوددهی را تنظیم کنید.
- هشدارها و توصیه‌ها را در تب **«هشدارها»** مشاهده کنید.
